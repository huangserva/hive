user 已在 Q10 拍板 D：本地 STT。HippoTeam 第一刀采用 openclaw 的"探测本地 CLI + 文件输出优先 + stdout fallback"思路，但不搬 sherpa / native binding：

- `src/server/local-stt.ts` 新增 `LocalSttProvider`
  - 探测顺序：`whisper-cli`（需要 `WHISPER_CPP_MODEL` 或 `HIVE_STT_WHISPER_CPP_MODEL` 指向 ggml 模型）→ Python `whisper`

- `src/server/feishu-transport.ts`
  - 飞书 audio message 下载后先写到临时 `.opus` 文件并调用 `LocalSttProvider`
  - 本地转写成功则直接复用 `[来自飞书语音]` inbound 注入链路
  - 本地不可用时保留原飞书 `speech_to_text.fileRecognize` 兜底；飞书免费版不支持时只 drop 当前语音并写日志

## 安装步骤（给真 E2E）

任选一种本地 STT CLI 放到 runtime 的 `PATH`：

### 方案 1：whisper.cpp `whisper-cli`（推荐，轻量）

```bash
brew install whisper-cpp
mkdir -p ~/.cache/whisper.cpp
curl -L -o ~/.cache/whisper.cpp/ggml-base.bin \
  https://huggingface.co/ggerganov/whisper.cpp/resolve/main/ggml-base.bin
export WHISPER_CPP_MODEL="$HOME/.cache/whisper.cpp/ggml-base.bin"
whisper-cli -m "$WHISPER_CPP_MODEL" -f /path/to/sample.wav -otxt -of /tmp/sample
```

确认 `whisper-cli` 和 `WHISPER_CPP_MODEL` 对启动 Hive runtime 的 shell 可见后，重启 runtime。

### 方案 2：Python OpenAI Whisper CLI（安装简单，资源更重）

```bash
python3 -m pip install -U openai-whisper
export HIVE_STT_WHISPER_MODEL=base
whisper /path/to/sample.wav --model "$HIVE_STT_WHISPER_MODEL" --output_format txt --output_dir /tmp
```

`HIVE_STT_WHISPER_MODEL` 可换成 `tiny` / `base` / `small`。中文准确率通常 `base` 起步更稳；模型越大越慢、越占磁盘。
