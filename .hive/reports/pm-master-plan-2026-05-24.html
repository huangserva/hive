作者：Orchestrator (Claude Opus 4.7)  ·  日期：2026-05-24  ·  状态：等 user 拍板

1.核心理念：人是 CEO，AI 是 COO

这套体系的最大价值不是"让 AI 自己 PM"，是让人和 AI 各做自己最擅长的事，并把协作的接缝处显式化：

人 CEO 你（huangserva）AI COO Orchestrator
擅长 方向判断、优先级、品味、说"不"、决策的 weight 执行、记录、跨 session 持久化、不偷懒、详细 review
不擅长 时间有限、记不住所有上下文、不愿做重复工作、不想被打扰 缺乏品味、容易死板、判断错优先级、看不到 future
体系给的 一个 dashboard 看全局 + 快速决策的 5 个按钮（同意 / 拒绝 / 等等 / 重排 / 删） 结构化的写作模板 + 强制 review 节奏 + 明确"什么时候必须问人"的规则

"不怕 token 消耗"翻译成设计语言：让 AI 做更多 deep work（每开 session 全量 plan-vs-actual review、自动写 ADR 草稿、自动 promote ideas、跨 workspace 体检），把 user 的时间花在看 AI 准备好的几个决策按钮上。

UI 关键设计

  • Topbar 取消独立 Todo / Plan 按钮，合并成一个 Cockpit 按钮。Cockpit 内含 Plan tab + 其他 5 个 tab + 底部 Todo 是 floating mini-drawer（保留 sprint 快查能力）
  • 底部"AI 准备好的待办行动"是核心——user 打开 Cockpit 第一眼就能看到"今天我要回答什么、批准什么"，不需要切 tab 找
  • tab badge 颜色语义：红色 = AI 急等 user / 橙色 = stale 需要刷新 / 灰色 = 普通计数
  • Baseline tab 显示 "stale" badge 当 baseline/*.md 上次更新距今超过 2 周（AI 自动比较 git log + 当前代码 diff 决定）

6.AI 主动行为清单（"不怕 token 消耗" 给的预算）

行为触发动作Token 成本
A1 · session 开场 review 每次 orch PTY 启动 (Layer A/B) 读 plan.md + tasks.md Done + baseline/*.md → 评估"实际 vs 计划差距、有没有跑偏" → 写到 .hive/handoff.html 顶部 ~10k tokens/次（baseline 全读）
A2 · 自动起草 ADR 对话中检测到 "我们决定..." / "选 X 而不是 Y" 句式 用 adr.template.md 起草一份草稿落 decisions/draft-YYYY-MM-DD-X.md，挂 open-questions.md 等 user 确认 ~5k tokens/次
A3 · 自动 promote ideas 每开 session 1 次 扫 inbox.md，对每条 idea 评估"现在跟 plan 哪个 milestone 有关 / 是否成熟"，成熟的写一条 Q 到 open-questions.md ~3k tokens/次
A4 · 周期性 baseline 体检 每完成 milestone git log 最近 commits → 检查 module-map.md 是否还准 → 不准就改 + 挂 Q 给 user 确认 ~8k tokens/次
A5 · archive audit 每月 1 次（或 plan.md / tasks.md 超过阈值） 看 active 文件大小 → 旧条目移到 archive/ → 更新 index ~2k tokens/次
A6 · cross-workspace drift 每天 1 次（或飞书远控时） 扫所有 workspace 的 baseline/risk-hotspots.md，发现共同 risk → 提议复用解决方案 → 挂 Q ~5k tokens/次/workspace

典型一天 token 预算估算：A1 + A2×2 + A3 + A5 = 25k tokens/session。按 Opus 4.7 输入 $15/M token + 输出 $75/M token 算，重 use 一天 1-3 sessions ≈ $0.5-2/天。换"user 不用记任何东西 + 决策被结构化呈现"——值。

8.实施计划：3 个 sub-phase

Phase C-1 · 文件层（runtime + 模板） ≈ 3 小时
新增 4 类文档 + 模板 + runtime 自动种子，无 UI，全靠约定。
  1. 新增 5 个模板字符串到 pm-templates.ts：
    • OPEN_QUESTIONS_TEMPLATE
    • IDEAS_INBOX_TEMPLATE
    • BASELINE_MODULE_MAP_TEMPLATE
    • BASELINE_RUNTIME_FLOWS_TEMPLATE
    • BASELINE_INDEX_TEMPLATE（其他 3 个 baseline 按需）
  2. 扩展 ensurePmDocs：创建 .hive/{open-questions.md, ideas/inbox.md, baseline/*.md, decisions/, research/, archive/}
  3. ORCHESTRATOR_RULES 加 6 节：
    • Open Questions: 不能直接派 → 必须挂这里等
    • Ideas: 灵感放 inbox，每 session 扫一次找成熟的
    • Baseline: 启动必读 baseline/*.md，代码变动时更新
    • Decisions: 检测到决策语言 → 起草 ADR
    • Archive: 月度 audit + 超阈值即移
    • Cross-workspace（如果有 N>1 个 workspace）：风险共享
  4. 关羽 实现 + 典韦 测试
Phase C-2 · Cockpit UI（统一 dashboard） ≈ 1.5 天
把 Plan + Todo + 新 4 类合并成一个 Cockpit drawer with 6 tabs + 底部行动栏。
  1. 新建 web/src/cockpit/ 目录：CockpitDrawer / CockpitTabs / 6 个 TabPanel
  2. 每个 tab 对应一个文件解析器（已有 plan-doc.ts，新增 questions-doc.ts / ideas-doc.ts / baseline-doc.ts / decisions-doc.ts / archive-doc.ts）
  3. WS 扩展：复用 /ws/plan，加 questions/ideas/baseline/decisions 多文件 watch（chokidar glob）
  4. 新 HTTP endpoint：GET /api/workspaces/:id/cockpit 返回汇总 ParsedCockpit
  5. 底部行动栏：从 open-questions.md 取 high priority + AI 标记的 promote 建议
  6. Topbar：移除 Todo / Plan 按钮（保留 Todo 作为 floating mini-drawer），加 🎯 Cockpit 按钮
  7. 关羽 实现（参考已有 PlanDrawer 范式） + 典韦 测试
Phase C-3 · AI 主动行为（A1-A6） ≈ 1 天
实现 6 个 AI 主动行为，让"AI COO"真起作用。
  1. A1 session-start review：在 ORCHESTRATOR_REMINDER_TAIL 加强制 review 指令 + 给 orch 一个 "session_started" 信号（runtime 检测 PTY 首次启动 → 注入 system msg "请跑 plan-vs-actual review"）
  2. A2 自动 ADR：system prompt 检测 "我们决定 / 选 X 不选 Y" 句式 → cp adr.template → draft
  3. A3 promote：system prompt 每开 session 提醒"先扫 inbox.md"
  4. A4 baseline 体检：每完成 milestone 在 system prompt reminder 触发
  5. A5 archive：cron 定时 / 文件大小检测
  6. A6 cross-workspace：飞书远控时附带
  7. 用 system prompt + reminder + runtime 触发组合实现（不需要新 worker / cron framework）
总工作量：约 3 天（C-1 3h + C-2 1.5d + C-3 1d）。比单独做 Phase C（3 天）节省 0，但效果是融合后体系完整，不留接缝。

9.跟人配合的关键时刻（5 个高价值 use case）

UC1 · 你早上看一眼"昨晚 AI 做了什么"

打开 Cockpit → 底部"AI 待办行动" 一眼看完 → 3 个红色 Q → 各点回答 → 派单触发 → 喝咖啡

价值：60 秒掌握项目状态。代替你以前要翻 chat history / git log / tasks.md

UC2 · AI 主动发现 plan 跑偏

orch 跑 A1 review 发现 M7 已 blocked 7 天没动 → 挂 Q "M7 要不要重新评估或调整 scope?" → 早上 user 看到 → 拍板

价值：drift 不再被忽略。代替"过了 1 个月 user 才意识到 M7 没做"

UC3 · 你出门，AI 全程透明

飞书发消息 → orch 派单 → worker 干完 → orch 自动写 milestone-review 挂 archive/ → 你回来看 archive 知道刚才发生了什么

价值：远程也能掌控感。配合 Phase 5 审批卡片 = 远控 + 透明 + 安全

UC4 · AI 用 baseline 替代死记硬背

session 重启 → orch 读 baseline/module-map.md 知道 src/server/* 各文件职责 → 派单时不需要先 grep → quality 上升

价值：跨 session 知识连续。代替"每次重启都要花 30min 重新熟悉代码"

UC5 · 你随手扔 idea，AI 帮你 promote

你忽然想到 "飞书加 /status 命令" → Cockpit Ideas tab 加一行 → 一周后 AI 发现 M11 feishu enhancement 起草 → 挂 Q 给你确认

价值：想法不会丢，但也不会立刻被实施（用 inbox 缓冲，避免"我随便说的话"被立刻当任务做）

UC6 · AI 帮你做"难写的"决策记录

对话中你说 "我们就用方案 B 吧"，orch 自动起草 ADR draft 含背景/选项/代价/结果 → 挂 Q "我起草了这个决策，要采纳吗" → 你点同意 → 永久归档

价值：决策不再口头消失。3 个月后 "我们为啥选 B" 不用回忆

10.风险 + 取舍

Risk影响缓解
AI 过度热情写文档，每个对话都生成 3 个文件 user 被淹没，反而不看了 system prompt 加约束："最多 1 个 ADR draft / session，最多 2 个 Q / session"
baseline 越写越多，AI 启动读得越来越慢 session 启动 latency 高，token 成本失控 baseline/*.md 每个限 200 行，超了要拆 + 归档
Cockpit drawer 因 tab 多变成"什么都看不进去" UX 倒退 底部"AI 准备好的待办行动栏"是核心，tab 是 drill-down。默认进入 user 看的是行动栏不是某个 tab
open-questions 答了 AI 不知道 AI 反复问 UI "回答"按钮触发明确事件 → 更新文件 + 派对应任务，不是 user 自己改文件 AI 不知道
跨 workspace baseline 有冲突 "这是哪个 workspace 的 risk?" baseline 严格按 workspace 隔离，cross-workspace AI 行为只读不写
3 天工作量真做完 user 可能体验失望（"还是要 review 这么多文档"） 沉没成本焦虑 分阶段 ship：C-1 文件层 3 小时先 ship，user 跑 1 周看是否真有用，再决定 C-2 UI 和 C-3 AI 行为
