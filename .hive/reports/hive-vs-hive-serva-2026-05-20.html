1.3.0 版本疑点

结论

1.3.0 是真实 npm 发布版本，不是本地源码字面量。用户能复现的来源是：

npm view @tt-a1i/hive dist-tags
# { "latest": "1.3.0" }

npm view @tt-a1i/hive versions
# ... "1.0.0", "1.1.2", "1.1.3", "1.1.4", "1.1.5", "1.2.0", "1.3.0"

npm view @tt-a1i/hive time
# "1.3.0": "2026-05-20T02:43:16.800Z"

本地两个仓库没有 1.3.0 源码版本：两边 package.json:3 都是 "version": "1.0.0"；rg "1\.3\.0" 只在锁文件依赖版本中命中 expect-type@1.3.0 和 string_decoder@1.3.0，不是 Hive 自身版本。

npm 1.3.0 对应什么

  • npm view @tt-a1i/hive@1.3.0 返回 gitHead: 8c3184b4fd250f04cdd0b77a02e5164319b1a121、tarball https://registry.npmjs.org/@tt-a1i/hive/-/hive-1.3.0.tgz。
  • 这个 gitHead 在当前 ~/development/hive 和 ~/development/hive-serva 都无法解析：git cat-file -e 8c3184...^{commit} 返回 128。
  • 公开 tt-a1i/hive 远端 heads 只有 main b6b4e1d、dev 27fd20f、feature/windows-native-support f7d9a91；git ls-remote 未找到 8c3184。
  • npm tarball 解包后 package.json:3 是 1.3.0，但 package.json:20 的 repository 指向 git+https://github.com/tt-a1i/hive-private.git。这说明 1.3.0 很可能从私有发布线构建，当前公开源码仓库尚未同步相同 commit。
  • tarball 中的 CHANGELOG.md:5 到 :80 记录 1.3.0 主要是 PWA/installable Hive、service worker、runtime offline page、Open Workspace 下拉改动、docked terminal panel、Add Member 模板入口调整。
  • tarball 确实包含 workspace shell runtime：dist/src/server/workspace-shell-runtime.js 存在；还包含 dist/src/cli/hive-update.js、dist/src/server/open-target-commands.js、web/dist/manifest.webmanifest 和 web/dist/logo.png。

tag / GitHub releases

  • 本地和公开 tt-a1i/hive tag 最高是 v1.0.0；没有 v1.3.0 或 1.3.0 tag。
  • huangserva/hive 本地和公开远端没有列出 tag。
  • gh release list 因未登录失败；改用 GitHub REST API 查询。tt-a1i/hive 公开 releases 只返回 v0.6.0-alpha.0，未见 1.3.0 release；huangserva/hive releases 为空。

为什么“npm 1.3.0，但代码没展示”

当前本地源码 checkout 还在 1.0.0 包版本线上，Web Topbar 读取的是本地 package.json 版本：web/src/version.ts:3 导出 APP_VERSION = pkg.version，web/src/layout/Topbar.tsx:49（hive-serva）或 hive :40 显示 v{version}。所以本地开发 UI 会显示 1.0.0，而 npm registry/latest 会提示 1.3.0。

之前那份对比报告哪里搞错了

如果旧报告说“hive-serva 是 hive 的功能增强分支，新增 workspace shell terminal”，方向是错的。正确链路是：

  1. workspace shell terminal 在共同历史中的 1e637b5 加入，并在 9363632 细化。
  2. hive-serva HEAD 就停在 9363632，所以它保留该功能。
  3. hive 在 9363632 之后通过 340869e 和 4032274 反向撤掉该功能。
  4. 因此看当前文件会误以为“serva 新增、hive 没有”；看拓扑和 revert 提交后，真实方向是“共同主线曾有，hive 后来删除，serva 保留旧状态”。
