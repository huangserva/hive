POST /api/settings/marketplace/import

Request:  { "slug": "security-auditor", "override_name": "我的审计员" }   // override_name 可选
Response 201:
{
  "slug": "security-auditor",
  "template": {
    "id": "<uuid>",
    "name": "我的审计员",
    "role_type": "reviewer",
    "description": "...",
    "default_command": "codex",
    "default_args": [],
    "default_env": {},
    "is_builtin": false
  }
}

Errors:
  400 — slug 缺失或未知 catalog slug
  403 — UI 凭证缺失（沿用 settings 路由统一鉴权）

5 · 验证

tests/server/marketplace-catalog.test.ts 13 测试全绿 + tests/server/settings-api.test.ts 4 测试无回归（共 17 测试）。

关键真测路径：

  • POST import 真发到 server → 落 sqlite role_templates 表 → 再 GET role-templates list 能读回；产品反就过不了
  • 不同 slug 不撞 BUILTIN id/name（unit 测试自动锁，加 sample 撞了会立刻 fail）
  • 每条 sample description 尾追 HippoTeam 纪律段（unit 测试锁，遗漏会 fail）
  • 同一 slug 多次 import 落多条记录（id 不同）
  • override_name 生效且空白 trim
  • 未知 slug / 缺 slug 都 400

6 · 分期路线 Phase1 Phase2 Phase3

Phase 1（本单，已完成）

  • catalog 数据模型 + 2 REST 路由 + 5 sample + 13 TDD 测试。
  • 无 Web UI 改动。curl 可走通：curl -b "hive_ui_token=..." -XPOST -H content-type:application/json -d '{"slug":"k8s-sre"}' http://127.0.0.1:4010/api/settings/marketplace/import

Phase 2（user 拍后）

  • AddWorkerDialog 内"📚 模板库"按钮 → 极简 modal（列 sample + 一键导入）；不照搬 upstream Drawer。
  • i18n 中英；导入成功后立刻在 AddWorker 角色下拉刷新可见。
  • 可能新增 confirm 提示如果重名（即使本单允许多实例，UX 上提示更友好）。

Phase 3（远期，看真实使用量）

  • 远程索引（GitHub raw / 自托管 catalog.json），按需 fetch + 缓存。
  • 分类树 / search / filter / preview。
  • 社区贡献入口（如果 catalog 真的有人贡献）。

8 · 剩余风险 / 阻塞

  • 无 UI 用户感知：Phase1 只能 curl 导入，普通 user 看不到；这是 PM 推荐的"先做最小"策略，用 ADR 锁住承诺 Phase2 上 UI。
  • sample 覆盖窄：5 条远不及上游 ~400；如果真的有用户要长尾 niche，Phase3 走远程索引最经济。
  • catalog 是代码不是配置：每加 sample 要发版；Phase3 起远程后此约束消失。
  • 导入策略允许同 slug 多实例：测试已锁；UX 上可能让用户困惑（同名两条）。Phase2 加 confirm。
  • 钟馗 review 待审：本单按派单顺序应该走 reviewer 审，配合 user 拍 Phase2 范围。
