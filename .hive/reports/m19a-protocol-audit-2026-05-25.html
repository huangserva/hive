HippoTeam Native App 稳定 API 盘点

结论：现有 web runtime 已经有足够数据源支撑 Expo/RN 只读 dashboard， 但它仍是“本机浏览器 UI 协议”，不是“LAN 原生客户端协议”。M19a 真正需要补的是 移动端认证、LAN 访问边界、紧凑 dashboard 聚合端点、只读 terminal/transcript 面。

Executive Summary

可复用

Cockpit aggregate、Workers list、Runs list、Runtime status、Workspace list 都已存在。 数据完整度足够做第一版只读 dashboard。

主要阻塞

HTTP 和 WS 都被 local-only guard 拦住；认证只支持浏览器 HttpOnly cookie。 手机走 LAN 时既过不了 Host/remote address，也没有 Bearer/device token。

建议方向

不让 Expo app 直接依赖 /api/ui/*。新增稳定 mobile dashboard API， 内部复用现有 parser/store，先只读，控制与 terminal input 留给 M19b/M19d。

Current Auth And Transport Boundary

Layer Current behavior M19a impact
HTTP local guard src/server/app.ts calls assertLocalRequest(request) before routing. LAN device requests are rejected unless the guard grows a paired-device path.
WS local guard Cockpit / Plan / Tasks / Terminal upgrade handlers call getLocalRequestRejection. Mobile realtime cannot connect over LAN today.
UI auth hive_ui_token HttpOnly cookie from GET /api/ui/session. Good for same-origin browser, weak fit for native client pairing and reconnect.
CLI agent auth x-hive-agent-id + x-hive-agent-token for team protocol. Should not be reused as mobile user auth; it grants worker/orchestrator semantics.

Prioritized Gap List

  1. Device auth / pairing: introduce paired-device credentials and bearer auth before allowing LAN access. Do not export the existing browser UI cookie as mobile auth.
  2. LAN-safe request gate: keep localhost default, but allow paired devices through a separate authenticated path for selected mobile endpoints and WS upgrades.
  3. Mobile dashboard aggregate: one endpoint and one WS stream that combine Cockpit, team, runs, runtime, and workspace metadata.
  4. Read-only terminal surface: expose transcript/run state before exposing raw PTY input/control channels.
  5. Schema/version contract: add schema_version and capability flags so the native app can survive server upgrades.

Recommended M19a Cut

For M19a, keep the app read-only and prove three things: it can pair/connect over LAN, it can render a native dashboard from a compact aggregate, and it can receive realtime dashboard updates. Do not implement mobile send/stop/terminal input until M19b/M19d capability gates are in place.
