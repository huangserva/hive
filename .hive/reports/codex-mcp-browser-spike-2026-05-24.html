Codex worker MCP browser E2E 能力

目标：让 Hive 的 Codex worker 在派单后能通过 MCP 打开真实浏览器、点击按钮、填写表单、截图和观察 DOM/可访问性快照，用于 Cockpit UI audit。

推荐结论

选型：@playwright/mcp@0.0.75

默认以 Codex preset args 注入，使用 npx -y @playwright/mcp@0.0.75 --headless --isolated --viewport-size=1440x1000。它本地运行、不需要 API key、能力覆盖 Cockpit E2E 所需的导航、点击、填写、截图、dialog 和 wait。Browserbase 暂不默认启用，因为它需要 API key 且 hosted endpoint 运行在云端。

Codex / Hive 集成

Codex CLI 0.133.0 支持 -c key=value 覆盖 ~/.codex/config.toml，并提供 codex mcp list。Hive 现有 preset 系统已经有 yoloArgsTemplate，会在 agent run bootstrap 时拼到 Codex 启动参数前面。因此本轮不扩 DB schema 字段，只刷新 builtin Codex preset 的 yolo args。

codex \
  --dangerously-bypass-approvals-and-sandbox \
  -c mcp_servers.playwright.command="npx" \
  -c 'mcp_servers.playwright.args=["-y","@playwright/mcp@0.0.75","--headless","--isolated","--viewport-size=1440x1000"]' \
  -c mcp_servers.playwright.startup_timeout_sec=30 \
  -c mcp_servers.playwright.tool_timeout_sec=60
