飞书 e2e 接入指南15 分钟从 0 到远控

为 huangserva 准备 · 2026-05-24 · 完成后你可以从飞书远控 hive workspace（含 Hermes 风格审批卡片）

0. 整体路径

  1. 飞书开放平台建自建应用 + 配权限 + 订阅事件（Step 1-4，~8 分钟）
  2. 拿凭证写本地文件（Step 5，~1 分钟）
  3. 发布应用版本（Step 6，~2 分钟）
  4. 重启 hive runtime 加载凭证（Step 7，~1 分钟）
  5. 飞书拉机器人到聊天 + 拿 chat_id（Step 8-9，~2 分钟）
  6. hive web UI 绑定 chat → workspace（Step 10，~1 分钟）
  7. 测试 e2e 闭环（Step 11，~3 分钟）
做完你可以在飞书 @bot 发消息，hive orchestrator 自动派 worker，结果回飞书；高风险动作弹审批卡片。
Step 1 建飞书自建应用 ~3 min
  1. 打开 https://open.feishu.cn/ 用你的飞书账号登录
  2. 顶部菜单 开发者后台 → 创建企业自建应用
  3. 填：
    • 应用名称：HippoTeam Orchestrator（任意，会显示在飞书联系人里）
    • 应用描述：HippoTeam workspace 远控机器人
    • 应用图标：随便传，不影响功能
  4. 提交。进入应用详情页面
Step 2 开启机器人能力 ~1 min
  1. 应用详情左侧菜单 添加应用能力
  2. 找到 机器人，点 添加
  3. 保持默认设置即可

这一步让你的应用成为"机器人"类型，可以加进群聊 + 私聊。

Step 3 配置权限 ~2 min

左侧菜单 权限管理 → 搜索并添加以下 4 个权限：

权限用途必需
im:message读取群聊和单聊消息✅
im:message:send_as_bot以机器人身份发消息✅
im:resource读 / 上传文件、图片资源✅（飞书 SDK 调 message.create 需要）
im:chat读群信息（拿 chat_id 等）✅

添加后页面顶部会出现"申请发布"提示，先不用申请，下面继续配。

Step 4 订阅事件 — 用 WebSocket 模式（关键） ~3 min

左侧菜单 事件订阅：

  1. 订阅方式选 使用长连接接收事件（即 WebSocket 模式）
    不要选"回调地址"——回调需要公网 IP 接收 HTTPS POST。你本地的 hive 没有公网 IP，必须用长连接（hive 主动拉取，飞书 SDK 已封装好）。
  2. 添加事件订阅，搜并添加 2 个事件：
    • im.message.receive_v1 — 接收用户发给机器人的消息
    • card.action.trigger — 接收用户点击审批卡片按钮

两个事件都加上。如果之后没看到 card.action.trigger 选项，可能搜 "card" / "卡片" 或检查机器人能力是否启用。

Step 5 拿凭证 + 写本地文件 ~1 min

左侧菜单 凭证与基础信息，记下：

  • App ID：形如 cli_abcdef123456
  • App Secret：32 位字符串

在你的 Mac 终端跑（不是 hive orch PTY 里，**单独开一个终端**）：

mkdir -p ~/.config/hive
cat > ~/.config/hive/feishu.json <<'EOF'
{
  "app_id": "cli_你的真实_app_id",
  "app_secret": "你的真实_app_secret"
}
EOF
chmod 600 ~/.config/hive/feishu.json
chmod 600 重要：app_secret 是凭据，等同于密码。文件权限 600 保证只有你自己读。不要 commit 这个文件到 git（hive 设计上凭证只在 `~/.config/hive/` 不在 workspace 内，已经隔离）。
Step 6 发布应用版本 ~2 min

飞书自建应用必须发布版本才能在群里工作。

  1. 顶部 版本管理与发布 → 创建版本
  2. 版本号填 1.0.0（任意），发布说明随便写
  3. 提交，状态会变成 已发布（个人自建应用免审批，立即生效）

企业版可能需要管理员审批 — 看你的飞书账号类型。

Step 7 重启 hive runtime 让它加载凭证 ~1 min

跟之前测 Cockpit 一样的重启操作。在你的 Mac 终端：

# 杀掉占 4010 的进程
lsof -ti:4010 | xargs kill -9

# 重启
cd ~/development/hive-serva
pnpm dev
注意：会杀所有 worker PTY（关羽 / 赵云 / 典韦 都变 stopped），需要重启后在 web UI 点 [Restart]。我（orchestrator）也会被杀，但 PM 体系 session-start nudge 会让新 session 自动 reload baseline + plan + open-questions。

启动后看日志确认凭证加载：

tail -f ~/.config/hive/logs/runtime-4010.log

应该看到：

[info] feishu credentials loaded app_id=cli_你的_app_id
[info] runtime started port=4010 ...
[info] feishu WSClient connected

如果看到 feishu credentials not configured → Step 5 文件位置 / 权限 / JSON 格式有问题，回去查。

Step 8 飞书拉机器人到聊天 ~1 min
  1. 打开飞书 app（手机或电脑都可以）
  2. 新建一个测试群（或单聊机器人）：
    • 方案 A（推荐）：单聊。直接搜你刚建的 "HippoTeam Orchestrator" 机器人 → 发消息 → 自动建私聊
    • 方案 B：群聊。新建群 → 加成员 → 选机器人

单聊比群聊更适合开始测试：消息不需要 @bot 就能触发，调试更顺。

Step 9 第一次发消息 + 拿 chat_id ~1 min
  1. 在飞书的私聊（或群里 @bot）发任意消息，比如 "hello"
  2. 立刻看 hive log（你打开的 tail -f）
  3. 应该看到： 
    [info] feishu inbound message chat_id=oc_xxxxxxxxxx sender=ou_yyyy
[info] feishu inbound dropped reason=no_binding chat_id=oc_xxxxxxxxxx
  4. 记下那个 oc_xxxxxxxxxx — 这是飞书 chat 的唯一 id，下一步用

"dropped reason=no_binding" 是正常的，因为还没绑 workspace。

Step 10 hive web UI 绑定 chat ↔ workspace ~1 min
  1. 打开 hive web UI http://127.0.0.1:5173
  2. 顶栏右上角应该看到一个 绿色 🪶 飞书指示灯（鼠标悬停 tooltip 显示 "connected" + app_id）
  3. 左侧 sidebar：当前 workspace（hive-serva）行右侧有 ⚙ 设置图标 → 点
  4. 弹出 dialog → 找 Feishu Integration 段
  5. 填 chat_id：oc_xxxxxxxxxx（Step 9 拿到的）
  6. chat name 可填 "测试群" / "我的私聊"（可选）
  7. 点 Bind

绑定成功后这个 chat 跟 hive-serva workspace 绑死，飞书来的消息会路由到这个 workspace 的 orchestrator。

Step 11 测试 e2e 闭环（3 个场景） ~3 min

测试 A: 简单对话

  1. 飞书发："你好，看一下当前 git status"
  2. 预期：
    • hive log 出现 feishu inbound message ... feishu inbound injected chat_id=oc_xxx workspace=hive-serva
    • orch PTY 内收到注入文本 [来自飞书 chat=oc_xxx ...] 你好，看一下当前 git status
    • orch 跑 git status（或派关羽跑）然后用 team feishu reply "状态如下..." 回飞书
    • 飞书收到回复

耗时一般 10-30 秒。orch 会先思考再调 team feishu reply。

测试 B: 派 worker 任务

  1. 飞书发："派关羽看一下今天的 commit 记录"
  2. 预期：orch 识别为派单 → team send 关羽 "..." → 关羽干完 team report → orch 用 team feishu reply 回飞书

测试 C: 审批卡片（高风险动作）

  1. 飞书发："删除 ~/Downloads/old-test-dir"（先 mkdir 一个真目录测试用）
  2. 预期：
    • orch 识别为高风险（含 rm 操作） → 调 team approve "删除 ~/Downloads/old-test-dir 目录" --risk high
    • 飞书收到红色审批卡片： 
      ┌──────────────────────────────────┐
│ 🟥 🤖 Hive 审批请求 · 高风险动作   │
├──────────────────────────────────┤
│ 动作                              │
│ 删除 ~/Downloads/old-test-dir     │
│                                  │
│ Workspace                         │
│ hive-serva                        │
├──────────────────────────────────┤
│   [✅ 允许]    [❌ 拒绝]            │
└──────────────────────────────────┘
    • 点 ✅ → 卡片变绿色"✅ 已允许 by @你 at HH:MM" + orch 收到 [Hive 系统消息：approval_id=xxx ALLOWED ...] → 继续删除 + 回复 "已删除"
    • 点 ❌ → 卡片变灰色 + orch 收到 DENIED → 用 team feishu reply "已撤销，请提供替代方案"
这是 Hermes 风格的"放心"功能。出门时手机上点一下就能批准 / 拒绝高危操作，user 全程透明可控。

常见问题排查

症状可能原因 + 修复
hive log 显示 feishu credentials not configured
  • 检查 cat ~/.config/hive/feishu.json 是否存在
  • 检查 JSON 格式（要双引号 + 没有 trailing comma）
  • 检查 chmod 600 后能不能读：cat ~/.config/hive/feishu.json
hive log 显示 feishu credentials invalid app_id 或 app_secret 拷错了，回 Step 5 重核
hive log 显示 WSClient error / 连不上
  • 检查飞书应用是否已发布版本（Step 6）
  • 检查权限是否齐 4 个（Step 3）
  • 检查事件订阅是 WebSocket 模式而不是回调（Step 4）
  • 网络问题：飞书 WS 走 open.feishu.cn:443，确认能访问
飞书发消息但 hive log 没动静
  • 检查机器人是否已加入聊天（私聊确认 / 群里见 bot 头像）
  • 群里要 @bot 才会触发（单聊不用）
  • 检查事件订阅 im.message.receive_v1 真订阅了
  • Topbar 🪶 指示灯是不是绿色，不是绿就是 transport 没起来
看到 inbound 但 reason=no_binding Step 10 没绑 chat_id 到 workspace。回去绑
orch 收到注入但没回 team feishu reply
  • orch 可能在做其他事，等几十秒
  • 看 orch PTY 输出是否有错
  • 检查 ORCHESTRATOR_RULES PROTOCOL.md 含 "team feishu reply" 规则（如果你重启过 4010，应该自动 regen 含完整规则）
审批卡片不弹
  • 检查事件订阅 card.action.trigger 真订阅了
  • 看 orch 是不是真调了 team approve（不是直接派单）—— 看 orch PTY 输出
  • 测试动作要 "明显高风险"（含 rm / git push / drop 等关键词），低风险动作 orch 不会触发审批

测完之后

保留飞书桥继续用

  • 凭证文件 ~/.config/hive/feishu.json 留着
  • 每次 pnpm dev 启动 4010 都会自动加载
  • chat 绑定持久化在 SQLite，重启不丢
  • 出门时直接发飞书消息就行

暂时关掉飞书桥

mv ~/.config/hive/feishu.json ~/.config/hive/feishu.json.disabled
# 重启 4010
# log 会显示 "feishu credentials not configured, transport disabled"
重新启用就把 .disabled 后缀去掉再重启。

彻底清掉

rm ~/.config/hive/feishu.json
# 在 hive web UI WorkspaceSettings 里 unbind 所有 chat
# 重启 4010
飞书应用本身可以在 open.feishu.cn 后台删除，或者保留备用。

测的过程中遇到问题

把以下信息发我，定位最快：

  1. 哪一步卡住
  2. hive log 最后 20 行：tail -20 ~/.config/hive/logs/runtime-4010.log
  3. 飞书 console 应用状态（已发布 / 权限齐 / 事件订阅在 WebSocket）
  4. orch PTY 内的最新输出（如果跟 orch 互动相关）

前几次很容易踩坑，飞书 OAS console 偶尔改 UI，你可能看到的菜单位置跟我写的略有不同——找关键字（机器人 / 权限 / 事件订阅 / 版本发布）就能找到。
