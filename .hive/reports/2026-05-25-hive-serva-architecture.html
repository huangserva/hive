Hive Serva Architecture

Local-first multi-agent workbench: React UI, Node runtime, SQLite persistence, PTY-backed CLI agents, team protocol, Feishu bridge, and workspace .hive PM documents.

UI Surface

  • React 19 + Vite render workspace sidebar, terminal panels, Cockpit, task graph, and plan drawers.
  • WebSocket clients subscribe to terminal, tasks, plan, and Cockpit snapshots.
  • HTTP client routes all mutations through local runtime APIs.

Runtime Core

  • Node runtime owns route registry, RuntimeStore facade, local request guard, auth tokens, and route-specific services.
  • Agent runtime starts real node-pty processes and tracks idle / working / stopped states.
  • Team protocol is asynchronous: send writes worker stdin, report injects back to orchestrator.

State Boundaries

  • SQLite persists workspaces, workers, messages, dispatches, runs, presets, templates, app state, and Feishu bindings.
  • Workspace .hive files are user-visible PM source of truth and feed Cockpit through chokidar.
  • Live PTYs, WebSocket clients, terminal mirrors, tokens, and approval ledger remain runtime-memory only.
